# AI Venture Lab Homepage Brief

## Goal

홈페이지는 단순한 소개 페이지가 아니라, 사용자가 이 제품을 **무엇으로 이해해야 하는지**를 바로 잡아주는 입구여야 한다.

- 이 제품은 랜딩 페이지보다 **실행 도구**에 가깝다.
- 사용자는 "AI가 아이디어를 대신 정리하고, 나는 판단만 보완한다"는 흐름을 10초 안에 이해해야 한다.
- 기본 사용자는 솔로 운영자이며, 협업은 옵션이어야 한다.

## Audience

1. 신사업/서비스 기획자
2. 운영자/매니저
3. 혼자 아이디어를 검증하고 MVP까지 밀고 싶은 사용자

## Homepage Structure

### 1. Hero

- 제품 한 줄 정의
- "실행 보드 열기" CTA
- 제품이 무엇을 자동화하는지 짧은 설명

### 2. Workflow

- 아이디어 정리
- 검증 판단
- 제작 준비
- 출시 후 학습

이 4단계는 제품 전체를 한눈에 설명하는 기본 흐름이다.

### 3. Why It Works

- AI가 먼저 초안과 질문을 만든다
- 한 사람이 끝까지 갈 수 있다
- 협업은 필요할 때만 붙인다

### 4. Deliverables

- 아이디어 브리프
- 검증 패키지
- PRD / MVP 범위
- 개발 태스크 보드
- 출시 판단 요약
- 학습 리포트

## Tone

- Agentic Workspace 구조
- Linear 스타일의 차분한 업무툴 톤
- 과한 "AI 제품" 연출 지양
- 섹션보다 행동과 결과를 앞세움

## Route Strategy

- `/` = homepage
- `/workspace` = execution board

로그인 이후 콜백 기본 진입점도 `/workspace`로 맞춘다.
