# Risk Register

| Risk | Area | Severity | Owner | Mitigation | Status | Beta Gate Disposition |
| --- | --- | --- | --- | --- | --- | --- |
| Personal data leakage | Privacy | High | Operator | Controlled beta must use beta-only/test data; do not intentionally collect real PII or customer-confidential data; keep launch evidence summary-only; define retention/deletion before broader launch | Accepted for beta scope | Accepted for controlled beta; blocks broader public launch until retention/deletion policy is finalized |
| Regulated advice claims | Legal | High | Operator | Do not present AI outputs as medical, legal, financial, employment, therapy, or other regulated advice; keep outputs as drafts and decision support only | Scoped out for beta | Non-blocking for controlled beta because regulated advice is out of scope |
| Weak idea validation | Product | Medium | Operator | Require idea brief, score gate, and decision log before build; beta remains a learning loop, not a product-market-fit claim | Accepted for beta scope | Non-blocking for controlled beta; blocks product launch claims |
| Secret exposure | Security | High | Operator | Keep `.env*` ignored except `.env.example`; keep secrets out of docs/chat/logs/screenshots; release check scans docs/templates for obvious secret patterns; rotate immediately on disclosure | Mitigated with monitoring | Non-blocking for controlled beta with rotation policy and scan guard |
| Telemetry ingest secret disclosed during smoke run | Security/Operations | High | Operator | Rotated `TELEMETRY_INGEST_SECRET` in Vercel Production, redeployed, and reran `pnpm smoke:telemetry` plus `pnpm smoke:telemetry:funnel` with the rotated value; keep future telemetry secrets out of chat, docs, logs, and screenshots | Mitigated | Non-blocking; external runtime rotation scope is `none_or_completed` for this decision |
| Beta smoke data leakage | Security/Operations | High | Operator | Use `docs/BETA_ENV_AND_SMOKE_BOUNDARY.md` and `docs/SMOKE_DATA_CLEANUP_RUNBOOK.md`; keep beta smoke names-only; retain disposable Auth/workspace fixtures for reruns; avoid screenshots with private data | Mitigated for controlled beta | Non-blocking; disposable smoke fixtures are `retained_for_rerun` or `not_applicable` |
| CI permission overreach | Security/Operations | Medium | Operator | GitHub Actions is limited to `.github/workflows/quality.yml` with `permissions: contents: read` and `pnpm quality:full`; no secrets, deploys, telemetry smoke, authenticated write smoke, production mutation, or Build Relay execution in default CI | Mitigated for default CI | Non-blocking while local `pnpm quality:full` and production smoke remain required release evidence |
| CI runner/runtime drift | Operations | Low | Operator | `Quality Gate` now opts into `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24` and `windows-2025-vs2026`; app build compatibility passed on Node 20 and Node 24 with the same read-only `pnpm quality:full` command while keeping no secrets, no deploys, and no production mutation | Monitoring with early compatibility matrix | Non-blocking for controlled beta; do not add `engines.node` or change Vercel Project Settings without a separate deployment-impact review |
| Missing denied-case RLS evidence | Security/Operations | High | Operator | RLS allowed/denied browser smoke passed with anonymous, disposable A, disposable B, and cross-workspace denied checks; keep evidence summary-only and rerun if RLS policies, fixtures, or workspace access code change | Mitigated | Non-blocking unless RLS or fixtures change |
| Unverified production RLS migration state | Security/Operations | High | Operator | Production posture was checked before denied smoke: required private-read migrations were present, old public-read policies were absent, and RLS was enabled on core tables; rerun posture checks when migrations or policies change | Mitigated | Non-blocking unless migrations or policies change |
| Agent drift | Operations | Medium | Operator | Keep `AGENTS.md` short; store durable docs in `docs/`; run harness and release checks | Monitoring | Monitor during beta |
| Broad authenticated write policy | Security | Medium | Operator | Operator ownership policies restrict updates and deletes to the row creator; add richer organization ownership before sensitive data | Accepted for beta scope | Non-blocking for current disposable beta; revisit before sensitive data |
| Built-in auth email delivery limit | Operations | Medium | Operator | Controlled beta signup uses a server-only Supabase Admin route that confirms beta accounts without sending email; configure custom SMTP before wider testing | Mitigated for controlled beta | Non-blocking for controlled beta; SMTP before wider invite flow |
| Unverified beta email ownership | Security/Privacy | Medium | Operator | The controlled beta signup path skips email confirmation to avoid the current SMTP blocker; keep access limited to invited/test users, monitor account creation, and restore verification before public signup | Accepted for controlled beta only | Blocks broader public signup until email confirmation or another verification path is configured |

Validation keywords: `risk_owner_recorded`, `beta_gate_disposition_recorded`, `high_risk_accepted_or_scoped_for_controlled_beta`, `risk_acceptance_recorded_for_controlled_beta`.

Launch closure notes:

- `Accepted for beta scope` does not mean the risk is gone; it means the operator accepts it for the restricted controlled beta.
- `Scoped out for beta` means the workflow must not make that claim or collect/use that data during the beta.
- External runtime rotation scope for the disclosed telemetry secret is recorded as `external_runtime_rotation_scope_none_or_completed` for this decision.

Validation keywords: `conditional_mitigation_not_launch_closure`, `external_runtime_rotation_scope_none_or_completed`, `high_risk_open_blocks_ship`.
